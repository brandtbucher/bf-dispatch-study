bf Dispatch Study
=================

This repository contains code for three different
[bf](https://en.wikipedia.org/wiki/Brainfuck) interpreters. The implementations
are all identical except for the dispatch mechanism used to execute the
instructions. The instructions are implemented in [`cases.h`](cases.h); each
interpreter defines its own `TARGET` and `DISPATCH` macros before `#include`'ing
the cases. Non-performance-critical code shared by all interpreters is defined
in [`common.c`](common.c).

### [`bf-loop`](bf-loop.c)

This interpreter uses a classic `switch` statement in a loop.

### [`bf-goto`](bf-goto.c)

This interpreter uses the non-standard
["labels as values"](https://gcc.gnu.org/onlinedocs/gcc/Labels-as-Values.html)
extension to the C language, and requires a compiler that supports it.

It moves the dispatch logic from a single shared location to the end of each
instruction, which allows branch prediction to work more effectively. It is
generally faster than `bf-loop`.

### [`bf-tail`](bf-tail.c)

This interpreter uses a tail-recursive dispatch mechanism. It requires a
compiler that supports
[`__attribute__((musttail))`](https://clang.llvm.org/docs/AttributeReference.html#musttail)
(it's recommended that the compiler also support the
[`__attribute__((preserve_none))`](https://clang.llvm.org/docs/AttributeReference.html#preserve-none)
calling convention as well).

It results in a similar branch-predictor-friendly design as `bf-goto`. However,
it is generally faster, since it also has the benefit of using smaller, simpler
functions that the compiler can optimize more effectively. For real-world
interpreters like [CPython](https://github.com/python/cpython/pull/128718) with
more complex control-flow graphs and higher register pressure, the effect of
this optimization is even more pronounced.


Building
--------

Each interpreter can be built by running `make <name>`. The resulting binary
takes one argument: a path to a program to interpret. For convenience, a sample
Mandelbrot program is provided in [`mandelbrot.bf`](mandelbrot.bf).

```console
$ make bf-loop
cc -O3 -Wall   -c -o bf-loop.o bf-loop.c
cc -O3 -Wall   -c -o common.o common.c
cc   bf-loop.o common.o   -o bf-loop
```

All three interpreters can be built with the default target, `make all`. Since
Clang 19 is currently the only compiler that supports all three dispatch
mechanisms, you may need to install it and modify the `CC` environment variable
appropriately:

```console
$ CC=clang-19 make
clang-19 -O3 -Wall   -c -o bf-loop.o bf-loop.c
clang-19 -O3 -Wall   -c -o common.o common.c
clang-19   bf-loop.o common.o   -o bf-loop
clang-19 -O3 -Wall   -c -o bf-goto.o bf-goto.c
clang-19   bf-goto.o common.o   -o bf-goto
clang-19 -O3 -Wall   -c -o bf-tail.o bf-tail.c
clang-19   bf-tail.o common.o   -o bf-tail
```

target. This will run each interpreter on the Mandelbrot program using the
